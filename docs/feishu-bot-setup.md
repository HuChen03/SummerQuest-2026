# 飞书机器人设置与部署说明

本文档是飞书机器人自愿实践的部署参考。创建、部署和验证机器人均不属于 A0 提交物、评分项或验收要求。

请直接使用仓库唯一保留的 [默认 Codex 飞书机器人](../starter/default_codex_bot/README.md)，不要从群聊或网上复制包含他人凭据的配置。

这套代码同时提供不调用模型的 `/ping <nonce>` 健康检查和基于 `PROFILE.md` 的 Codex 自动问答。同学可以按 [默认实现与自定义说明](default-codex-feishu-bot.md) 修改或完全替换为自己的方案。

公开仓库不提供 AIGW 地址。需要运行默认实现时，请从组织内飞书课程文档获取地址，并只通过本地 `AIGW_BASE_URL` 环境变量注入。

## 命名

```text
OpenMOSS-暑期集训-2026-<同学真名>
```

## 示例功能

- 启用机器人能力。
- 使用长连接或实验室认可的方式接收消息，不要求开放公网回调端口。
- 收到 `/ping <nonce>` 后在 10 秒内回复 `pong <nonce>`。
- 进程在 SSH 断开后继续运行。
- 重启进程后仍能完成 `/ping` 验证。

实现语言和框架不限。建议使用飞书官方 SDK。机器人源代码和部署配置不要求提交到 GitHub，也不需要为 A0 提交验收材料。

## 服务器部署要求

1. 使用普通用户运行，不以 root 身份运行机器人。
2. 使用独立 Python virtual environment 或等价的隔离环境。
3. 凭据通过环境变量或权限为 `600` 的本地文件注入。
4. 使用 `systemd --user`、`tmux`、`supervisord` 等方式保持进程运行。
5. 日志不得打印 App Secret、Token、完整消息内容、环境变量或其他凭据。

个人服务器的主机名、IP、SSH 用户名、内部域名和日志原文不写入公开 GitHub。如需自行记录部署过程，应放在组织内文档中并完成脱敏。

## GitHub 不登记机器人信息

学生不需要在 GitHub 中填写机器人名称、App ID、权限、源代码、模型、部署状态或消息记录，也不需要在 A0 `README.md` 或补充文档中登记机器人信息。

## 禁止提交的信息

以下内容不得写入 GitHub，也不得复制到个人主页或作业飞书文档正文：

- App Secret
- Verification Token
- Encrypt Key
- Webhook Secret
- Access Token 或 Refresh Token
- Cookie、密码、SSH 私钥
- 内部服务凭据
- AIGW 地址和其他内部服务地址

如果凭据曾经进入 Git commit，即使随后删除文件也视为已经泄露：立即停止使用、轮换凭据并联系助教处理 Git 历史。

## 可选自检记录

如需记录自愿实践过程，只保存最小必要、已经检查和脱敏的信息：

- 机器人能力、最小权限和可用范围的文字记录；不要上传凭据页面截图。
- 服务器进程 active 状态、PID、运行时长和程序名，不显示环境变量或完整参数。
- 自行执行 `/ping <nonce>` 与 `pong <nonce>` 的测试记录。
- 重启前后的验证结果。
- 不含密钥的部署步骤和故障排查记录。

相关文档设置为组织内公开，不得开启互联网公开访问。
